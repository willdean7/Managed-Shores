This folder contains all datasets used in the Managed_Shores project.

## Case Study Folder Contents

Each case study folder contains the following subfolders and files:

    [site]/
    ├── cosmos/              # Raw CoSMoS hazard rasters and vector files
    ├── derived/             # Processed datasets output by analysis scripts
    ├── redfin_2025-XX.csv   # Raw Redfin MLS property data (not redistributed)
    └── redfin_df.csv        # Cleaned and processed property data

### `cosmos/`
Raw coastal hazard data from the USGS Coastal Storm Modeling System (CoSMoS):
- **Flood sites** (Carpinteria, King Salmon, Stinson Beach): CoSMoS v3.0 — flood inundation depth, wave height, flood duration at discrete SLR increments and storm return periods
- **Cliff sites** (Isla Vista, Pacifica): CoSMoS v3.1 — cliff retreat projections under "Let It Go" and "Hold The Line" management scenarios

> Source: <https://www.usgs.gov/apps/coastal-storm-modeling-system-cosmos>  
> License: Public domain (U.S. Government work)

### `derived/`
Processed outputs generated by the analysis pipeline. Key files:

| File | Description | Generating Script |
|------|-------------|-------------------|
| `cosmos_hazard_metrics.csv` | Property-level hazard extractions at discrete SLR levels | `extract_cosmos_metrics_unified.R` |
| `cosmos_annual_hazards.csv` | Annual hazard timeseries interpolated from CoSMoS using OPC 2024 SLR projections | `interpolate_cosmos.R` |
| `retreat_schedule_baseline.csv` | Property-level T* and buyout prices under baseline parameters | `cosmos_valuation_v3.R` |
| `retreat_schedule_sensitivity.csv` | Government-side economics across sensitivity parameter combinations | `cosmos_valuation_v3.R` |
| `mc_damage_distributions_baseline.csv` | Full Monte Carlo damage distributions (1,000 iterations × property × scenario) | `cosmos_valuation_v3.R` |
| `community_stats_baseline.json` | Site-level summary statistics | `cosmos_valuation_v3.R` |

---

## Usage Notes

- Do not edit raw data files directly — create derived versions in the `derived/` subfolder
- Scripts must be run separately for each site by updating `case_name` at the top of each script
- Raw Redfin MLS files are not included in the Dryad data deposit due to redistribution restrictions
