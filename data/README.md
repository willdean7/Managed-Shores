## Case Study Folder Contents

Each case study folder contains the following subfolders and files:

    [site]/
    ├── cosmos/              # Raw CoSMoS hazard rasters and vector files
    ├── derived/             # Processed datasets output by analysis scripts
    ├── redfin_2025-XX.csv   # Raw Redfin MLS property data
    └── redfin_df.csv        # Cleaned Redfin property data

### `cosmos/`
Raw coastal hazard data from the USGS Coastal Storm Modeling System (CoSMoS):
- **Flood sites** (Carpinteria, King Salmon, Stinson Beach): CoSMoS v3.0 — flood inundation depth, wave height, flood duration at discrete SLR increments and storm return periods
- **Cliff sites** (Isla Vista, Pacifica): CoSMoS v3.1 — cliff retreat projections under "Let It Go" and "Hold The Line" management scenarios

> Source: <https://www.usgs.gov/apps/coastal-storm-modeling-system-cosmos>  
> License: Public domain (U.S. Government work)

## Missing Values

Missing values in all files are represented as `n/a`. Two categories of missing data are present:

**1. Beyond-horizon properties (`retreat_schedule_baseline_*.csv`, `retreat_schedule_sensitivity_*.csv`, `timing_sensitivity_baseline.csv`)**

Properties where projected hazard exposure never triggers retreat within the 2026–2100 planning horizon are assigned `n/a` for all economic output columns, including `retreat_year`, `mc_mean_year`, `mc_median_year`, `mc_q05_year`, `mc_q25_year`, `mc_q75_year`, `mc_q95_year`, `buyout_price`, `leaseback_revenue`, `gov_net_cost`, `gov_outcome`, `npv_rent_annual`, `npv_damages_annual`, `npv_net_annual`, and associated damage distribution columns. For cliff sites (Isla Vista, Pacifica), this means cliff distance never reaches the 10-meter safety threshold by 2100. For flood sites, this means cumulative flood damages never exceed rental income within the planning horizon.

**2. Incomplete Redfin listing fields (`cosmos_hazard_metrics.csv`)**

Columns inherited from Redfin MLS exports — including `SOLD DATE`, `HOA/MONTH`, `DAYS ON MARKET`, `NEXT OPEN HOUSE START TIME`, `NEXT OPEN HOUSE END TIME`, `STATUS`, `SOURCE`, and `MLS#` — are not available for all properties and are not used in the economic analysis. Missing values in these columns are `n/a` and reflect incomplete listing data in the original MLS records.

## Usage Notes

- Do not edit raw data files directly — create derived versions in the `derived/` subfolder
- Scripts must be run separately for each site by updating `case_name` at the top of each script
- Raw Redfin MLS files are not included in the Dryad data deposit due to redistribution restrictions
